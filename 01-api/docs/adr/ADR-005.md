# ADR-005: Pest sobre PHPUnit directo

## Contexto

El proyecto necesita un framework de testing para PHP. PHPUnit es el estándar de facto en el ecosistema PHP y viene integrado con Laravel. Sin embargo, su sintaxis es verbosa (clases, métodos anotados, setUp/tearDown) y propensa a código boilerplate.

El equipo busca una experiencia de testing más fluida que fomente escribir más tests con menos fricción, manteniendo compatibilidad total con PHPUnit y el ecosistema Laravel.

## Decisión

Usar **Pest ^3.0 como framework principal de testing**, con las siguientes reglas:

- **Prohibido**: Escribir tests con sintaxis PHPUnit directamente (`class FooTest extends TestCase`).
- **Obligatorio**: Todo test usa sintaxis Pest (`it()`, `test()`, `expect()`).
- Pest corre sobre PHPUnit por debajo, por lo que toda la infraestructura de PHPUnit (cobertura, reportes, CI) sigue funcionando.
- Se usa el plugin `pest-plugin-laravel` para integración nativa con Laravel.

La suite incluye:

| Tipo | Sintaxis Pest | Prohibición |
|---|---|---|
| Unit tests | `it()`, `test()` | Clases anónimas PHPUnit |
| Feature tests | `it()`, `test()` con `get()`, `post()` | Clases que extienden `TestCase` directamente |
| Architecture tests | `test()->arch()->expect('src')->not->toUse('Illuminate')` | — |

## Alternativas consideradas

| Alternativa | Razón de descarte |
|---|---|
| PHPUnit directo | Sintaxis verbosa, más boilerplate. El proyecto es nuevo y no tiene tests legacy que migrar, por lo que no hay razón para mantener compatibilidad hacia atrás. |
| PHPT | Demasiado simple, no apto para tests de integración con Laravel. |
| Codeception | Sobredimensionado para este proyecto: agrega otra capa de abstracción sobre PHPUnit sin beneficio claro frente a Pest. |

## Consecuencias

**Positivas:**
- Tests más legibles y concisos: `it('validates email format', fn () => ...)`.
- Las architecture tests (`test()->arch()`) son una ventaja diferencial para mantener Clean Architecture.
- Compatibilidad total con PHPUnit: cobertura, CI, reportes funcionan igual.
- Pest v3.x tiene integración nativa con Laravel 13.

**Negativas:**
- Dependencia adicional (`pestphp/pest`, `pestphp/pest-plugin-laravel`).
- Desarrolladores que solo conocen PHPUnit necesitan aprendizaje inicial.
- Algunos plugins de PHPUnit pueden no ser compatibles (verificar en cada actualización).
